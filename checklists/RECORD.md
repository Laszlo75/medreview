# RECORD — routinely-collected health data (extends STROBE)

Source: Benchimol et al., *PLoS Med* 2015. Use **alongside `STROBE.md`** for
studies using registries / administrative / EHR data (UKTR, NHSBT, HES, SRTR,
CPRD). RECORD adds/expands the STROBE items below. Reviewer: mark met /
partial / unmet with location. The starred items are the highest-yield gaps
in registry work.

## Title & abstract
- **RECORD 1.1** — Type of data and the database(s) named in title/abstract.
- **RECORD 1.2** — Geographic region and timeframe stated.
- **RECORD 1.3** — If linkage across databases, this is stated.

## Methods — participants (STROBE 6)
- **RECORD 6.1 \*** — The **codes / algorithms** used to define the study
  population are listed in full; if not possible, an explanation is given.
  *(The single most common failing — the cohort must be reproducible from the
  routine data. Demand the extraction definition.)*
- **RECORD 6.2** — Validation of the population-selecting codes/algorithms
  described, with reference, if done.
- **RECORD 6.3 \*** — A **flow diagram** of how records became the study
  population (records → linkage → eligibility → final n).

## Methods — variables (STROBE 7)
- **RECORD 7.1 \*** — Complete list of codes/algorithms/definitions used to
  classify exposures, outcomes, confounders and effect modifiers; validation
  status of each stated.

## Methods — data sources/measurement (STROBE 8) & linkage
- **RECORD 12.1** — Linkage technique and any measures of linkage quality
  described (if applicable).

## Results — data access & cleaning (STROBE 13)
- **RECORD 13.1** — Extent to which investigators had access to the database
  population used to create the study cohort; data-cleaning steps and numbers
  excluded at each step.

## Discussion — limitations (STROBE 19)
- **RECORD 19.1 \*** — Implications of using data **not collected for
  research**: misclassification, unmeasured/residual confounding, changes in
  coding over time, missingness patterns, and data-access limitations.

## Other
- **RECORD 22.1** — Whether the study protocol, the linked database, and the
  extraction code are available to others, and how to access them.
